Backend

`browser_onnx` is the default backend. `Load And Prepare` preloads the browser runtime in the current tab to avoid first-synthesis model load latency.

The default packaged model path in this build is `models`.

If the packaged model directory is missing, `Load And Prepare` will download the Hugging Face ONNX assets into browser-managed storage automatically and switch the backend to that managed path.

`Read Page`, `Reload Voices`, and `Save Voice` now reuse the same auto-download path, so you no longer need to click `Load And Prepare` first just to fetch missing models.

Packaged extension models and browser-managed downloads do not require `Allow access to file URLs`. Only manually entered external local file paths need that permission.

Browser ONNX PoC

This opens the CPU-only browser ONNX page. It supports built-in voices, uploaded voices, editable text, and warm-up loading without the Python server.

Server Connection

Changing the host or port here updates future requests immediately. You do not need to reload the extension.

In `browser_onnx` mode, the uploaded prompt audio is encoded locally in the browser and stored in extension storage. In `server` mode, it is uploaded to the Python server like before.

Generation Options

Max TTS Batch Size defaults to 1 in this build. Set it to 0 to re-enable auto batching. Buffered generation keeps chunk order and decodes codec sub-batches no larger than the current TTS batch. Realtime mode keeps output order and uses the smallest active chunk-group width among auto batching, Max TTS Batch Size, and Max Codec Batch Size.

This app is CPU-only. CPU Threads maps to `torch.set_num_threads` for that request. It defaults to 4, and 0 means auto using the server default thread count.

CPU Attention Backend options are synchronized with the official CPU-only app: `model_default`, `sdpa`, and `eager`. On CPU, `model_default` currently resolves to `eager`.

Under the browser_onnx backend, Enable WeTextProcessing is still not executable today. The cached .fst files can be reused as assets, but browser inference still needs a browser-side OpenFst/WeText runtime before those graphs can actually run.

Under the server backend, both sampled modes still map to regular do_sample=true. The extra sampled-mode split is only implemented by the browser_onnx runtime.

Fixed Sampled Frame runs a dedicated browser ONNX fast path with host-provided random numbers and graph-internal sampling. Its fixed sampling hyperparameters are: text temp 1.0 / top-p 1.0 / top-k 50, audio temp 0.8 / top-p 0.95 / top-k 25 / repetition penalty 1.2.

When Enable normalize_tts_text is checked, extracted paragraph text is still normalized again before it is sent into browser ONNX inference.
