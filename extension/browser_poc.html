CPU-only Browser ONNX PoC

Nano Reader Browser PoC

This page runs the same browser ONNX runtime used by the extension backend. It supports built-in voices, uploaded voice cloning, editable text, non-streaming synthesis, and realtime streaming decode on local CPU.

The Load And Prepare button preloads tokenizer, ONNX sessions, and warm-up runs so synthesis does not pay the first-load penalty. The packaged default models live under extension/models/MOSS-TTS-Nano-100M-ONNX and extension/models/MOSS-Audio-Tokenizer-Nano-ONNX and load without file-URL permission. If that packaged directory is missing, Load And Prepare will download the Hugging Face ONNX assets into browser-managed storage automatically. Only manually entered external local paths require Allow access to file URLs.

Text Normalization

Realtime vs non-streaming is selected by the two run buttons below. This PoC applies Sampling Mode, Enable normalize_tts_text, Initial Playback Delay, CPU Threads, and Voice Clone Max Text Tokens.

Under the browser_onnx backend, Enable WeTextProcessing is still not executable today. The cached .fst files can be reused as assets, but browser inference still needs a browser-side OpenFst/WeText runtime before those graphs can actually run.

Fixed Sampled Frame uses a dedicated fast path with host-provided random numbers and graph-internal sampling. Its fixed hyperparameters are text temp 1.0 / top-p 1.0 / top-k 50, audio temp 0.8 / top-p 0.95 / top-k 25 / repetition penalty 1.2.

Attention Backend, Max TTS Batch Size, and Max Codec Batch Size are shown here for popup parity and shared settings sync, but the current browser ONNX PoC runtime does not consume them yet.
